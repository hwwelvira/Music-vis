# 🎵 Music Dimension Visualizer - 前端交互分析系统

### 音乐流派与声学特征高维交互可视化驾驶舱

> **项目定位**：基于 Spotify 经典音乐数据集（1960-2020）的多维声学与文化特征交互分析系统。
> **技术栈**：`React 18` + `Vite` + `ECharts 5`
> **数据源**：`public/data.json`（已进行离线预处理与聚类归档）

---

## 🧭 核心功能板块与技术实现

系统围绕**流派归属**、**声学特征**、**情感空间**与**词频文化**四大维度，实现了高度关联的双向联动交互：

### 1. 🌊 流派结构与声学特征透视仪 (`ToggleableChordRadar`)
* **双子星流派探索视角 (Toggleable Sunburst & Chord)**：
  * **双层旭日图视角 (Sunburst)**：呈现多级流派的嵌套放射层级关系。节点亮度引入**流行度自适应亮度因子**，低流行度节点自动降低亮度（最高淡化 `55%`），实现视觉分级。
  * **弦图视角 (Chord Diagram)**：支持一键切换为弦图视图。弦图刻画了不同细分流派在数据集中的**跨流派杂交/融合关联度 (Chords Link)**，弦线粗细代表两流派在歌曲中的共存度或演变亲和度。
* **旭日/弦图与雷达图的深度联动机制**：
  * **悬停临时联动 (Hovering)**：当鼠标在旭日图节点或弦图流派边缘悬停时，右侧声学雷达图（展示 **可舞性、能量、声学度、情感效价 (Valence)、活跃度、器乐度、言语度** 7 个维度）会即时更新绘制该流派的声学雷达多边形（呈**淡紫色虚影**）。此为即时交互态，移开即自动重置。
  * **锁定核心联动 (Locking/Click)**：当点击旭日图节点或弦图流派时，系统会**锁定该流派的声学雷达多边形（呈深紫色实线）**。此操作将常驻雷达图，并在页面右侧和底部的其他面板中同步激活该流派的歌曲过滤与排序，作为探索基准。

### 2. 📊 主界面流派声学对比与相似度分析
* **锁定对比面板 (主界面右侧上方)**：
  * 支持用户自由点选 2 至 4 个任意流派（包含大分类与子细分类），在主界面的对比雷达图中绘制重叠多边形。
  * **极简声学相似度计算模型**：基于 7 维声学特征向量，通过两两流派对的**平均绝对误差 (MAE)** 距离倒数进行加权相似度折算：
    $$\text{Similarity} = \max(0, 1 - \text{MAE} \times 1.6) \times 100\%$$
  * 相似度计算结果在简约的灰色标签条内展示，提供“高度相似 / 中度相似 / 差异显著”的分级判定。

### 3. 🌀 流派专属画像悬浮窗 (Draggable Context Modal)
* **开启方式**：点击旭日图或弦图的子流派节点触发。
* **数据构成**：
  * **高频歌词词云**：展示该流派歌曲中前 20 位高频词汇。
  * **历代流行度变迁折线图**：绘制该流派从 1960 至 2020 年的流行度走向，并提供“本流派”、“所属大类”以及“全球大盘均值”的三线对比，用于时间变迁上的趋势洞察。
* **交互机制**：
  * **阻尼拖动 (Draggable)**：鼠标按住小窗口头部可在页面内平移，支持随时叉掉关闭。

### 4. 🔤 歌词词频多维特征透视仪 (Word Feature Analyzer)
* **开启方式**：在“流派画像”小窗口中点击词云的任何一个歌词标签触发。
* **计算逻辑**：
  * **流派亲和度**：遍历系统包含的所有流派，统计该词在各流派中的出现频次，排序并提炼其主要关联流派。
  * **情感声学指纹**：提取各关联流派的 7 维声学特征，以词频为权重进行**加权平均**，计算出该词在整个音乐库中的虚拟声学特征向量。
* **可视化方案 (无重复图表)**：
  * **流派特征：南丁格尔玫瑰图 (Nightingale Rose Chart)**：花瓣的半径和面积双重编码该词在不同流派中的浓度分布。
  * **情感特征：极坐标单轴放射柱状图 (Polar Radial Bar Chart)**：以极坐标放射线条的长度表征加权后的 7 维声学特征强度。
* **交互机制**：支持窗口关闭与鼠标任意位置的拖动平移。

### 5. 🌌 KMeans 情感空间星云图 (`ScatterBrushChart` / Voronoi map)
* **开启方式**：点击主界面功能入口，以弹窗全屏模式呈现。
* **情绪空间分布**：横轴表示**能量 (Energy)**，纵轴表示**愉悦度 (Valence)**。
* **Voronoi 背景网格剖分**：计算 5 个情感聚类中心（质心），在 50×50 的网格平面上按最近距离自动剖分，生成五种色彩的情感边界区域（狂热释放、轻松愉快、伤感静谧、阳光活力、迷幻张力）。
* **优先级排他去重算法**：当大流派（Parent）与旗下子细分类同时被勾选为对比系列时，大流派系列会自动剔除子流派的歌曲，防止因为散点物理坐标重合而在 ECharts 中发生系列遮挡遮盖 Bug。
* **极简降噪高亮联动**：鼠标悬停在弹窗顶部的流派卡片上时，该流派的散点保持尺寸不变，而**其他非悬停流派的散点透明度瞬间降至 `0.08`（淡化为虚影）**。移开后通过 React 虚拟 DOM 秒级重绘恢复（`0.88`），无任何 ECharts Action 残留 Bug。
* **星云散点图与右侧歌曲列表的高阶协同**：
  * **流派勾选协同**：弹窗顶部的流派卡片组不仅过滤了散点图，也**作为右侧歌曲表格明细的实时过滤网**，表格仅滚动展示被勾选流派的高热度歌曲（前100首）。
  * **单曲双向定位联动**：在右侧表格中点击某首特定单曲时，左侧星云图会**精确反向定位该曲粒子坐标，触发红色气泡呼吸扩散的高亮动画效果**，帮助用户在庞大的星云中精确锁定该单曲的情感方位。

### 6. 🎧 单曲多维声学特征深度解析悬浮窗 (Song Detail Modal)
* **开启方式**：在主界面歌曲明细表格中，点击任何一首歌曲触发。
* **交互机制**：支持窗口关闭 `✕`，支持按住窗口头部进行**全平移无阻尼拖拽**，允许用户放置在屏幕任意位置以防遮挡。
* **数据对比与可视化**：
  * 系统计算该单曲的 7 维声学特征。
  * 采用**多边形重叠对比雷达图**（或条形偏差图），将该单曲的特征多边形与**所属 KMeans 聚类均值特征多边形**、**全球大盘声学均值特征多边形**进行三线偏差比对。
  * 帮助用户直观、量化地评估该单曲相对于大盘和所属流派在声学本质上的“特异性”（如它是更狂热、更可舞还是更静谧）。

### 7. 📋 多维属性编码单曲列表 (`SongTable`)
* **多层信息编码**：
  * 列表单曲左侧配备 **流派色圆点**。
  * 紧随其后配备 **KMeans 聚类色小圆点**，无需冗余文字即可传达歌曲流派与聚类双重属性。
  * 附带彩色流派小胶囊与热度百分比进度。
* **框选极速响应**：毫秒级响应主界面散点情绪图的 Brush 框选操作，过滤显示区域内的高热度歌曲。

## ⚙️ 本地部署与运行指南

### 🛠️ 环境依赖
* **Node.js** (推荐 `v16` 或 `v18+`)
* **npm**

### ⚛️ 启动前端服务
1. **进入项目目录**：
   ```bash
   cd music/spotify-vis
   ```
2. **安装依赖**：
   ```bash
   npm install
   ```
3. **启动开发服务器**：
   ```bash
   npm run dev
   ```
   启动后在浏览器打开终端提示的地址（默认 `http://localhost:5173/`）。

4. **构建生产版本**：
   ```bash
   npm run build
   ```

## 🗃️ 数据预处理与离线损耗优化
为了保证大屏可视化在复杂多维过滤、Voronoi 图实时渲染以及高频计算时的极端流畅度，原始 20MB 的 Spotify 歌曲数据集已采用 **Python 进行了离线预处理**：
1. **去噪与中文汉化**：通过定制的高防卫音乐字典，完成流派汉化、大类归纳及正则化过滤。
2. **高频歌词提取**：预先提取各流派的前 20 位歌词特征高频词，并对年代（1960-2020）均值进行整理，存入 `genre_details` 节点。
3. **降维与前端缓存**：保留核心声学数据直接内置于 `public/data.json`，在前端零开销、开箱即用。
